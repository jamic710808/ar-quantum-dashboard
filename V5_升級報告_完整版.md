# AR 量子智慧戰情室 V4 → V5 升級報告

## 升級概述

本文件詳細記錄應收帳款智慧戰情室從版本 V4.0 升級至 V5.0 的完整流程與技術變更。V5 版本代號「量子進化」，象徵從傳統帳務管理邏輯進化為具有預測能力與業務賦能的智慧決策系統。

---

## 一、升級背景與動機

### 1.1 V4 版本的局限性

V4.0 Ultimate 版本雖然已具備完整的帳齡分析、客戶評級、催收管理等功能，但在以下面向存在不足：

- **靜態分析為主**：僅能呈現當下帳款狀態，無法預測未來趨勢
- **缺乏業務視角**：應收帳款責任無法歸因到特定業務人員
- **風險判斷片斷**：僅有帳齡老化資訊，缺少客戶信用生命週期觀點
- **決策支持薄弱**：缺少 What-If 模擬與情境分析能力

### 1.2 V5 版本的定位

V5 「量子智慧戰情室」將定位從「帳務管理工具」提升為「信用風險預測與業務賦能平台」，強調三個核心能力：

1. **預測能力**：從歷史數據學習，預測未來帳款變化
2. **組合思維**：將授信視為投資組合進行優化
3. **歸因分析**：將帳款狀況與業務行為連結

## 二、功能模組升級

### 2.1 Tab 5：客戶生命週期（Customer Lifecycle）

#### 升級內容

| V4 舊有功能 | V5 新增功能 |
|-------------|-------------|
| 靜態帳齡表格 | 信用生命週期四象限散點圖 |
| 客戶基本資訊 | 信用成熟度演進折線圖（多客戶比較） |
| 單一客戶評級 | 生命週期策略矩陣表（授信額度 + 預警閾值） |

#### 核心視覺化

**四象限散點圖**
- X 軸：帳齡（天數）
- Y 軸：帳款金額
- 點大小：客戶營收貢獻
- 顏色分群：新客戶期（綠）/ 成長期（藍）/ 成熟期（紫）/ 衰退期（紅）

**信用成熟度演進圖**
- 多條折線追蹤不同客戶的信用評分變化
- 時間跨度：過去 12 個月
- 可切換比較：最多 5 個客戶同時顯示

**生命週期策略矩陣**
- 橫軸：授信額度建議（低/中/高）
- 縱軸：預警閾值（寬鬆/標準/嚴格）
- 9 宮格呈現對應策略文字說明

#### 設計理由

傳統帳齡分析是「快照」，只能看到「現在欠多少」；生命週期分析是「影片」，可以看到「客戶信用如何演變」。這對於長期信用管理決策至關重要。

### 2.2 Tab 6：授信組合矩陣（Credit Portfolio）

#### 升級內容

將金融投資組合的概念引入授信管理，重新定義客戶分類邏輯。

| V4 舊有功能 | V5 新增功能 |
|-------------|-------------|
| 客戶風險分級（A/B/C/D） | 2×2 組合矩陣（業績 × 風險） |
| 帳齡分布長條圖 | 目前配置 vs. 建議配置甜甜圈對比 |
| 壞帳準備金估算 | 最佳化收益預測（預計改善金額） |

#### 2×2 組合矩陣定義

```
                    高風險
         ┌─────────────┬─────────────┐
         │   穩定型    │   問題型    │
  低業績 ├─────────────┼─────────────┤
         │  觀察客戶   │  催收優先   │
         ├─────────────┼─────────────┤
         │  潛力客戶   │  明星客戶   │
  高業績 ├─────────────┼─────────────┤
         │   成長型    │   核心客戶   │
         └─────────────┴─────────────┘
                    低風險 ←─────────→ 高風險
```

#### 甜甜圈對比圖

- 左側甜甜圈：當前授信配置比例
- 右側甜甜圈：建議配置比例
- 中間差異金額：預估改善收益（$340,000）

#### 設計理由

授信決策不應只看風險，也要看「CP值」——同樣的授信額度，給高業績客戶的邊際收益較高。V5 將這個概念量化並視覺化。

### 2.3 Tab 7：預測引擎（Forecast Engine）

#### 升級內容

這是 V5 最核心的創新功能，將應收帳款管理從「事後分析」提升為「事前預測」。

| V4 舊有功能 | V5 新增功能 |
|-------------|-------------|
| 歷史帳齡趨勢圖 | 30/60/90 天預測帳齡趨勢 |
| - | What-If 模擬（利率 + 催收強度） |
| - | 客戶還款機率熱力圖 |
| - | 三種情境分析（樂觀/基準/悲觀） |

#### What-If 模擬控制項

- **年化利率滑桿**：0% - 15%，影響客戶付款意願
- **催收強度係數**：0.5x - 2.0x，影響催收頻率與力度
- **模擬期間**：1-6 個月滾動預測

#### 預測區段視覺

- 歷史資料區段：實線
- 預測區段：虛線 + 背景漸層陰影（表示不確定性區間）
- 30/60/90 天里程碑：垂直參考線標註

#### 還款機率熱力圖

- X 軸：未來 12 週
- Y 軸：Top 10 重要客戶
- 顏色：還款機率（紅 < 30% < 黃 < 70% < 綠）

#### 三情境說明

| 情境 | 利率變化 | 催收強度 | 壞帳率預測 |
|------|---------|---------|-----------|
| 樂觀 | -2% | 1.5x | -15% |
| 基準 | 持平 | 1.0x | 基準 |
| 悲觀 | +3% | 0.5x | +25% |

#### 設計理由

V4 只能告訴你「昨天發生了什麼」，V5 告訴你「明天可能發生什麼」並讓你模擬「如果我這樣做，結果會怎樣」。

### 2.4 Tab 8：業務歸因（Sales Attribution）

#### 升級內容

將應收帳款責任歸因到業務人員維度，實現「誰的客戶誰負責」的管理邏輯。

| V4 舊有功能 | V5 新增功能 |
|-------------|-------------|
| 客戶所屬業務欄位 | 業務員帳齡績效排行條形圖 |
| 帳齡分布統計 | 多業務員信用管控雷達圖對比 |
| - | 業務員 × 風險等級熱力矩陣 |

#### 業務員帳齡績效排行

- 排序維度：平均帳齡 / 壞帳率 / 催收成功率
- 圖表類型：水平條形圖
- 標註：與團隊平均的偏差百分比

#### 信用管控雷達圖

- 6 個維度：帳齡控制 / 壞帳率 / 催收效率 / 客戶成長 / 額度使用率 / 預警準確度
- 比較：最多 3 位業務員同時顯示
- 數值：標準化為 0-100 分

#### 業務員 × 風險等級熱力矩陣

- X 軸：業務員
- Y 軸：風險等級（低/中/高/呆帳）
- 顏色：客戶數量或帳款金額

#### 設計理由

應收帳款不應只是財務部門的責任。當業務員能看到「我的客戶帳款狀況」並與績效掛鉤時，會更有動機在銷售階段就做好信用風險把關。
